Dev workstation setup - All the pretty things

This is not to be confused with setting up Linux servers but this is actually about setting up your environment to a state where you can make it your own. Give it a little bit of style here and there, but most importantly make it yours.

I will be sharing a few of my favourite resources and tips to get your environment in a good direction.

1. New terminal

I recommend trying out new terminals on your workstation. If you are on windows, I highly recommend installing WSL. This allows you to install a subset system of Linux directly ontop of your Windows machine! As always, feel free to check out the resources on this section if you need some sort of guidance or tutorial on how to get this up and running.

If you're on MacOS, I recommend switching from the regular terminal into iTerm2. This terminal has many useful features and comes with OhMyZsh pre-installed.

Another useful tool or terminal that I love using is HyperJS. This terminal is an eletron-based terminal built on HTML Javascript. A very customizable terminal which can be a good alternative specially for Windows power users.

2. Themes

Freel free to find something that makes your workspace pop. There's plenty of themes out there but one of my favourites is Dracula Theme. This one has installation instructions to basically anything you want to throw it at. Let it be VScode, Slack, VIM, etc. Perfect for cases where you want to keep one same aesthetic across your apps and devices.

Feel free to look at the resources section if you'd like to learn a bit more on this.

3. Dotfiles

It is important to start building your own dotfiles. Dotfiles are configuration files for various programs, and they help those programs manage their functionality. What sets them apart from regular files and directories is their prefix. Dotfiles are named that way because each file and directory starts with a dot ( . )

You'll find a curated list of dotfiles here which has configuration for several different programs where you can start off and mix and match to your liking. However, feel free to find other resources on this if you'd like to dig deeper on customization and if you want to apply further workflows and shortcuts on your terminal. Really, your imagination is your limit here.

4. Software Installation

A short list of the programs I install on the machine using apt or brew

- VSCode 
- awscli 
- containerd.io
- docker
- docker-ce 
- iTerm2
- terraform

Feel free to experiment and share any new findings with the team!

5. Fonts

If you're someone who enjoys customizing everything, including fonts... then I do recommend getting a nice-to-read font that includes font ligatures!

Jetbrains Mono has a very decent one that is completely free. You can set this up to be included into your IDEs such as VScode as well as your terminal needs. There's plenty of different fonts out there, some are free and some are paid but feel free to experiment and choose whatever works best for you.
